# 订单类型详解

> 选对订单类型，是控制交易成本和控制风险的第一步。

---

## 一、基础订单类型

### 1. 市价单（Market Order）

以当前市场上最好的价格立即成交。

```
特点：
✅ 保证成交（几乎瞬间）
❌ 不保证成交价格
⚠️ 在市场剧烈波动时，成交价可能与预期相差很大
```

**适用场景**：流动性好的大盘股、紧急平仓、不在意几美分价差的时候。

**警告**：不要在盘前/盘后使用市价单！流动性差时滑点可能巨大。

---

### 2. 限价单（Limit Order）

指定一个价格，只有在等于或优于该价格时才成交。

```
例：AAPL 当前 $180.50
下限价买单：买入 100 股 @ $180.00
→ 只有在价格跌到 $180.00 或以下时才会成交

下限价卖单：卖出 100 股 @ $181.00
→ 只有在价格涨到 $181.00 或以上时才会成交
```

```
特点：
✅ 精确控制成交价
❌ 不保证成交（可能错过机会）
```

**适用场景**：日常交易的主力订单类型。

---

### 3. 止损单（Stop Order）

当价格触达止损价时，自动转为市价单。

```
例：持有 AAPL @ $180，设置止损卖单 @ $175
→ 当价格跌到 $175 时，自动以市价卖出
```

```
特点：
✅ 自动止损，不需要盯盘
⚠️ 止损触发后用市价成交，极端行情下成交价可能远低于止损价
```

**适用场景**：持仓保护，离开电脑时防止大亏。

---

### 4. 止损限价单（Stop Limit Order）

止损触发后转为限价单（而非市价单）。

```
例：持有 AAPL @ $180
止损价：$175
限价：$174
→ 跌到 $175 触发止损，但只在 $174 以上才卖出
→ 如果暴跌超过 $174，可能卖不掉
```

```
特点：
✅ 比纯止损单更好的价格控制
❌ 暴跌时可能无法成交（保护失效）
```

---

## 二、高级订单类型

### 5. 追踪止损单（Trailing Stop）

止损价会跟随价格上涨而自动上移，但价格下跌时不动。

```
例：持有 AAPL @ $180，设置追踪止损 $5（5美元差价）
→ 涨到 $190，止损价自动上移到 $185
→ 涨到 $200，止损价自动上移到 $195
→ 跌回 $195，触发卖出
```

**适用场景**：趋势跟踪，让利润跑的同时锁定收益。

---

## 三、订单有效期

| 类型 | 说明 |
|------|------|
| **DAY** | 当日有效（默认），收盘未成交则自动取消 |
| **GTC**（Good Till Cancel） | 取消前一直有效，通常 90 天自动取消 |
| **IOC**（Immediate or Cancel） | 立即成交可成交部分，其余取消 |
| **FOK**（Fill or Kill） | 必须全部成交，否则全部取消 |

---

## 四、推荐使用策略

| 场景 | 推荐订单类型 |
|------|-------------|
| 日常交易 | 限价单 |
| 紧急平仓 | 市价单 |
| 止损保护 | 止损限价单 |
| 趋势跟踪 | 追踪止损单 |
| 不在电脑前 | GTC 限价单 + 止损单 |

---

## 五、本系统的下单命令

```bash
# 限价买单
ibkr-stock order buy 100 AAPL --type limit --price 180.00

# 止损卖单
ibkr-stock order sell 100 AAPL --type stop --price 175.00

# 带止损的限价买单
ibkr-stock order buy 100 AAPL --type limit --price 180.00 --stop-loss 175.00
```
