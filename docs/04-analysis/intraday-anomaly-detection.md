# 盘中异常检测算法详解

> 关联: [Market Data Pipeline](../MARKET_DATA_PIPELINE.md)

---

## 为什么需要分钟级数据？

日线的致命盲区——它只记录一天的开盘、最高、最低、收盘：

```
某股票 6月18日:
  日线: 开 $300 · 高 $305 · 低 $270 · 收 $295

看起来: 跌了 1.7%，还行
实际情况:
  09:32  闪崩到 $270 (-10%)
  09:50  反弹到 $290
  10:30  回到 $298
```

日线把盘中 -10% 的暴跌"吞掉"了。只有分钟线才能发现这个级别的异常。

---

## 一、Z-Score 闪电崩盘检测

### 原理

Z-Score = (当前价格 - 近期均价) / 近期标准差

```
公式: Z = (P - μ₂₀) / σ₂₀

  P    = 当前分钟收盘价
  μ₂₀  = 过去20分钟的均价
  σ₂₀  = 过去20分钟的标准差
```

### 为什么有效？

股价短期波动近似正态分布。Z < -3 意味着当前价格偏离均值超过 3 个标准差——这在统计上发生的概率 < 0.3%。

```
正常情况:    Z 在 -1 ~ +1 之间 (68%)
小幅波动:    Z 在 -2 ~ +2 之间 (95%)
异常:        Z < -2.5 或 Z > +2.5
极端异常:    Z < -3   或 Z > +3  ← 触发告警
```

### 检测窗口

| 窗口 | 灵敏度 | 适用场景 |
|:--:|------|------|
| 5分钟 | 极高 | 容易误报，噪音多 |
| 10分钟 | 高 | 短线交易 |
| **20分钟** | 适中 | **推荐** — 平衡灵敏度和可靠性 |
| 60分钟 | 低 | 适合检测持续性下跌 |

### 代码示例

```python
def zscore_detection(prices, window=20):
    """检测闪电崩盘"""
    series = pd.Series(prices[-window:])
    z = (series.iloc[-1] - series.mean()) / series.std()
    
    if z < -3.0:
        return f"闪电崩盘 Z={z:.1f}σ"
    elif z < -2.5:
        return f"异常下跌 Z={z:.1f}σ"
    return None
```

## 二、ROC (Rate of Change) 反转检测

### 原理

ROC = (当前价 - N分钟前价格) / N分钟前价格

```
公式: ROC_N = (P_t - P_{t-N}) / P_{t-N} × 100%

  5分钟 ROC:  (P_now - P_5min_ago) / P_5min_ago
  15分钟 ROC: (P_now - P_15min_ago) / P_15min_ago
```

### 反转信号

```
跌后反弹 = 左侧抄底信号:

  ROC_15 < -5%  (过去15分钟跌超5%)
  且 当前价 > 5分钟前价格  (开始回升)
  → 可能见底反弹
```

```
涨后回落 = 追高风险信号:

  ROC_5 > +5%  (过去5分钟涨超5%)
  → 短期过热，追高风险极大
```

## 三、VWAP 偏离检测

### 原理

VWAP (Volume-Weighted Average Price) = 成交量加权均价，反映"市场平均成本"。

```
公式: VWAP = Σ(P_i × V_i) / ΣV_i

  P_i = 第i分钟的价格
  V_i = 第i分钟的成交量
```

### 为什么重要？

- **机构交易员**用 VWAP 衡量自己的成交质量
- 价格偏离 VWAP → 意味着短期过度涨跌 → 大概率回归
- VWAP 是日内交易的"公允价值线"

### 检测

```
当前价 > VWAP × 1.03  → 高于均价 3%，短期超买，注意回落
当前价 < VWAP × 0.97  → 低于均价 3%，短期超卖，可能反弹
```

## 四、成交量异常检测

### 原理

正常成交量遵循一定规律（开盘大、午间小、收盘大）。突然的成交量放大代表"异常事件"。

```
量比 = 当前5分钟成交量 / 近20日同期5分钟均量

量比 > 3x: 有资金在行动
量比 > 5x: 大资金进出，需关注方向
```

### 量价配合判断

| 价格 | 成交量 | 含义 |
|------|:--:|------|
| 上涨 | 放量 | 多头主动买入，看涨 |
| 上涨 | 缩量 | 上涨乏力，可能回落 |
| 下跌 | 放量 | 空头主动卖出，看跌 |
| 下跌 | 缩量 | 下跌动能衰竭，可能反弹 |

## 五、夜盘/盘前异常检测

夜盘和盘前交易量小、流动性差，不能用OHLCV K线。改用**价格快照**：

### 夜盘异动

```
30分钟内价格涨跌 > 3% → 触发告警

原因: 夜盘通常波动极小（< 1%），3% 已经是大事件
```

### 盘前趋势

```
用线性回归拟合过去60分钟的价格快照:

  斜率 > 0.05%/分钟 → 持续上涨趋势
  斜率 < -0.05%/分钟 → 持续下跌趋势

这比单点比较更可靠——能区分"真趋势"和"噪音波动"
```

## 六、多时间框架共振（二期）

### 原理

单时间框架信号可能是假信号。多框架同时发信号 → 可靠性大幅提升。

```
1分钟框架: 检测超短期波动（闪电崩盘）
3分钟框架: 确认短线趋势
5分钟框架: 短期方向
15分钟框架: 中期方向
30分钟框架: 持仓周期参考

五框架同向 + 量放大 → 高质量交易信号
```

### 共振判断

```
所有框架的 SMA 方向一致 (都在价格上方或下方)
且 至少3个框架出现量放大
且 Z-Score 绝对值 > 2
→ 强信号，可信度高
```

## 七、参数调优建议

| 参数 | 默认值 | 更激进 | 更保守 |
|------|:--:|:--:|:--:|
| Z-Score 阈值 | -3σ | -2.5 | -3.5 |
| 检测窗口 | 20分钟 | 10分钟 | 30分钟 |
| 量比阈值 | 5x | 3x | 8x |
| VWAP 偏离 | 3% | 2% | 5% |
| ROC 反转 | 跌5%+升2% | 跌3%+升1% | 跌8%+升3% |

**建议从保守开始**，避免噪音过多导致告警疲劳。根据实际触发频率逐步调优。
