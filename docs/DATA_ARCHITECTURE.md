# 股票数据架构 — 分钟级数据 + 短时异常检测

> 状态: 计划中 | 关联: [项目计划书](../PROJECT_PLAN.md)

---

## 问题

日线无法检测盘中异动：

```
AAPL 今天:
  开盘 $300 → 10分钟跌到 $270 (-10%) → 30分钟反弹到 $295 (-1.7%)

日线看: 开盘 $300, 当前 $295, 跌了 1.7%，没什么大不了
真实情况: 盘中暴跌 10%，但因为没有分钟数据，完全看不见
```

---

## 方案

### 三级数据存储

```
┌─────────────────────────────────────────────────────────────┐
│ 第一层: 实时分钟线 / 快照 (当日缓存)                            │
│   盘中: AAPL_2026-06-18_1min.parquet   (1分钟 OHLCV K线)     │
│   夜盘: AAPL_2026-06-18_extended.parquet (1分钟价格快照)      │
│                                                              │
│   用途: 盘中实时异常检测                                       │
│   生命周期: 当日保留                                          │
├─────────────────────────────────────────────────────────────┤
│ 第二层: 日聚合 (Parquet 永久保留)                              │
│   AAPL_1day.parquet                                          │
│   用途: 趋势分析、支撑阻力、退出策略、回测                        │
│   生命周期: 永久                                                │
├─────────────────────────────────────────────────────────────┤
│ 第三层: 实时快照 (内存, 60秒TTL)                               │
│   富途 overnight_price / bid / ask / volume                   │
│   用途: 当前价格、盘口价差                                     │
│   生命周期: 60秒后过期                                        │
└─────────────────────────────────────────────────────────────┘
```

### 数据流

```
盘中 9:30-16:00 (1分钟 OHLCV K线):
  IBKR reqHistoricalData(bar_size="1 min") → AAPL_0618_1min.parquet
  每根K线: timestamp, open, high, low, close, volume

夜盘 20:00-4:00 + 盘前 4:00-9:30 (1分钟快照):
  富途 get_market_snapshot → overnight_price → AAPL_0618_extended.parquet
  每条: timestamp, price, bid, ask, volume

所有时段 → 当日分钟数据 → 短时异常检测 (Z-Score/ROC/VWAP/量异常)
                                         │
                                         ├── 触发 → 📧 邮件告警
                                         └── 正常 → 继续积累

收盘后 (16:00 ET):
  分钟数据 → 聚合 OHLCV → 追加到 1day.parquet
  分钟数据 → 可选保留或删除
```

### 盘中 vs 夜盘

| | 盘中 (9:30-16:00) | 夜盘/盘前 |
|------|:--:|:--:|
| 数据来源 | IBKR/富途 K线 | 富途快照 |
| 数据内容 | OHLCV 完整K线 | 价格 + 成交量 |
| 频率 | 每分钟 1 根 | 每分钟 1 次快照 |
| 异常检测 | 6种算法 | 3种算法 |

---

## 短时异常检测算法

### 盘中 (1分钟K线)

| 检测项 | 算法 | 分钟线需求 | 触发条件 |
|--------|------|:--:|------|
| **闪电崩盘** | 20分钟 Z-Score | 20根 | Z < -3σ |
| **VWAP 偏离** | 当日累计 VWAP | 全部 | 偏离 > 3% |
| **成交量异常** | 5分钟量 vs 20日均量 | 5-20根 | > 5x 均量 |
| **反转信号** | 15分钟 ROC + 反弹 | 15根 | 跌>5%后回升>2% |
| **突破确认** | 开盘区间突破 + 量 | 30根 | 突破 + 量>2x |
| **跳空回补** | 开盘 vs 昨收 vs 15分钟走势 | 15根 | 跳空后回补50%+ |

### 夜盘/盘前 (1分钟快照)

| 检测项 | 算法 | 数据需求 | 触发条件 |
|--------|------|:--:|------|
| **夜盘异动** | 30分钟价格变化率 | 30个快照 | 涨跌 > 3% |
| **盘前跳变** | 最近价 vs 昨收 | 1个快照 | 偏差 > 5% |
| **盘前趋势** | 线性回归斜率 | 60个快照 | 斜率 > 0.05%/分钟 |

## 存储估算

```
盘中 1分钟K线 (390分钟):
  390行 × 6字段(OHLCV+时间) × 8字节 ≈ 19KB/天

夜盘+盘前快照 (540分钟, 20:00-9:30):
  540行 × 4字段(价格+买卖+量) × 8字节 ≈ 17KB/天

合计每只股票每天: ~36KB
65只 × 365天 ≈ 850MB/年
65只 × 30天  ≈ 70MB
```

完全可以本地存储，不需要删除历史。

## 异常检测流程

```
monitor daemon 每次扫描:
  1. 拉分钟数据 (如果当日文件不存在 → 新建；存在 → 追加新分钟)
  2. 跑短时异常检测 (Z-Score, ROC, VWAP...)
  3. 有异常 → 立即邮件告警
  4. 收盘后 → 分钟数据聚合为日线 → 写入永久存储
```

## 实现计划

### 新增文件（2个）

**`src/data/minute_store.py`** — 分钟线存储器
- `MinuteStore(symbol)` → 管理单只股票当日分钟数据
- `append(timestamp, price, volume)` → 追加
- `get_series(minutes=20)` → 获取最近 N 分钟
- `close_day()` → 聚合日线，写入永久存储
- 存储: `data/minutes/AAPL_2026-06-18.parquet` + `AAPL_2026-06-18_extended.parquet`

**`src/analysis/anomaly.py`** — 短时异常检测器
- `AnomalyDetector` 基于分钟数据
- 9 种检测算法（6 盘中 + 3 夜盘）
- 返回 `AnomalyAlert` 列表

### 修改文件（2个）

**`scripts/monitor_daemon.py`**
- 每次扫描追加分钟数据
- 扫描完成后跑异常检测
- 异常告警加入邮件

**`src/cli/main.py`**
- `analyze` 命令显示当日分钟走势概要

### 不修改
- `price_validator.py` — 实时快照逻辑不变
- `stock_data.py` — 日线存储不变
- `exit_strategy.py` — 基于日线，不变
