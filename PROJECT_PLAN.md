## 一、项目概述

基于 Interactive Brokers（盈透证券）API，构建一个模块化的 Python 股票交易助手。核心目标：

- **降低手动交易的操作门槛**：CLI + 通知即可完成日常操作
- **智能风控**：时政分析 + 日内交易限制（PDT） + 仓位管理
- **量化策略实验平台**：策略基类统一接口，方便统计物理方法论扩展
- **学习友好**：内置金融知识文档，边交易边学习

---

## 二、技术选型

| 层面 | 选型 | 理由 |
|------|------|------|
| **IBKR 接口** | `ib_insync` | 社区最活跃的 Python IBKR 封装，异步支持好 |
| **IBKR 客户端** | **IB Gateway**（推荐） | 比 TWS 更轻量，适合策略长期运行和无人值守 |
| **数据存储** | SQLite + Parquet | SQLite 存交易记录；Parquet 存历史K线 |
| **技术指标** | `pandas-ta` | 覆盖 200+ 技术指标 |
| **NLP / 情感分析** | `transformers` + FinBERT | 财经新闻情感分析，评估时政事件影响 |
| **微信通知** | 企业微信机器人 Webhook | 免费、支持 Markdown，推送交易信号和告警 |
| **远程计算** | rsync + SSH（远程服务器） | 大规模回测/参数优化同步到远端执行 |
| **配置管理** | `python-dotenv` + YAML | 敏感信息 .env，策略参数 YAML |
| **Web 界面** | Streamlit（二期可选） | 快速搭建监控面板 |

## 四、功能模块详解

### 4.1 连接管理 (`core/connection`)

```
连接策略：
- 默认使用 IB Gateway（端口 4002 live / 4001 paper）
- 同时兼容 TWS（端口 7497 live / 7496 paper）
- IB Gateway 优势：更轻量、无弹窗、适合长时间无人值守
- 心跳检测 + 断线自动重连（指数退避）
- 连接状态对外暴露，供通知模块使用
```

### 4.2 市场数据 (`data/`)

```
- 实时行情：reqMktData() 推送流数据
- 历史K线：reqHistoricalData() + 本地 Parquet 缓存
- 增量更新策略：按 contract+barSize 建索引，只拉新增数据
- 规避 IBKR pacing limit（~60次/10分钟）
```

### 4.3 交易执行 (`trade/`)

```
- 下单：市价单 / 限价单 / 止损单 / 止损限价单 / 追踪止损
- 订单管理：查询、修改、取消
- 持仓查询：当前持仓、盈亏、成本价
- 账户摘要：净清算值、可用资金、保证金

📊 交易记录存储（recorder.py）：
- 每笔订单存入 SQLite，字段包括：
  时间 | 股票 | 方向(买/卖) | 数量 | 价格 | 订单类型 |
  成交状态 | 是否日内交易 | 盈亏 | 策略名 | 备注
- 支持按日期/股票/策略等维度查询
- 提供统计分析接口：胜率、盈亏比、月度总结
```

### 4.4 风控模块 (`trade/risk.py`) ⭐ 核心

```
🛡️ 多层风控体系：

第一层：PDT 日内交易限制（硬限制）
  - 跟踪过去 5 个交易日内的日内交易次数
  - 日内交易定义：同一股票同日买入又卖出（或相反）
  - ≤ 2 次：正常交易，每次日内交易后推送提醒
  - = 2 次：橙色预警，下单前强制二次确认 + 微信推送警告
  - ≥ 3 次：红色硬阻止，拒绝所有日内交易订单
  - 例外：紧急平仓按钮（需双确认 + 记录原因）

第二层：仓位与资金管理
  - 单只股票最大仓位：总资金的 X%（可配置，默认 20%）
  - 总持仓上限：总资金的 Y%（可配置，默认 80%）
  - 单笔最大亏损：Z%（可配置，默认 2%）

第三层：熔断机制
  - 日亏损熔断：当日累计亏损超过 N%，停止交易
  - 连续亏损熔断：连续 M 笔亏损，暂停交易 K 分钟
  - 周亏损熔断：当周累计亏损超过 P%，停止交易

第四层：时政风险调整（见 4.7）
```

### 4.5 策略引擎 (`strategy/`)

```
策略基类接口：
  class Strategy(ABC):
      def on_bar(self, bar) -> Signal | None: ...
      def on_tick(self, tick) -> Signal | None: ...
      def on_news_sentiment(self, sentiment) -> float: ...  # 接收情感分数
      def on_order_filled(self, trade): ...

内置策略：
  1. 双均线动量策略（SMA 金叉/死叉）
  2. 布林带均值回归
  3. （预留）统计套利——配对交易
```

### 4.6 回测模块 (`backtest/`)

```
本地回测：小规模、单策略快速验证
远程回测（远程服务器）：
  - 大规模参数扫描（网格搜索/贝叶斯优化）
  - 多策略并行回测
  - 通过 Makefile: make remote-backtest
  - 结果回传本地查看
```

### 4.7 时政信息分析 (`news/`) ⭐ 新功能

```
工作流程：

  新闻源                NLP 分析              风控调整
  ─────────────────────────────────────────────────
  RSS 订阅   ──┐
  财经 API   ──┼──→ fetcher.py ──→ sentiment.py ──→ impact.py
  Twitter/X  ──┘        │               │              │
                       筛选去重      FinBERT 情感     风险等级调整
                      关键词匹配      打分 (-1~+1)     仓位建议变更
                                                      推送重要新闻

新闻源（可配置）：
  - RSS: Yahoo Finance, Reuters, CNBC, Bloomberg
  - API: NewsAPI, Finnhub（免费额度可用）
  - 关键词过滤：美联储/利率/CPI/非农/地缘/制裁/财报...

情感分析：
  - 默认使用 FinBERT（金融领域专用 BERT，HuggingFace）
  - 轻量备选：VADER（规则引擎，无需 GPU）
  - 输出：单条新闻情感得分 [-1, 1] + 聚合市场情绪指数

风险调整（impact.py）：
  - 高负面影响新闻 → 提高风控等级
    · 建议减仓/暂缓开仓
    · 推送微信告警
  - 重大突发事件 → 触发紧急模式
    · 立即推送所有重要新闻摘要
    · 建议暂停新交易
    · 允许紧急平仓（PDT 例外通道开放）
```

### 4.8 通知系统 (`notify/`) ⭐

```
📱 企业微信机器人推送（主通道）：

  推送内容：
  ├── 🔴 紧急（实时）
  │   ├── 重大时政事件 + 影响评估
  │   ├── PDT 次数预警（第 2 次日内交易后）
  │   ├── 止损触发 / 熔断触发
  │   └── 连接断开 / 异常告警
  ├── 🟡 重要（实时）
  │   ├── 订单成交确认
  │   ├── 市场情绪剧烈变化
  │   └── 持仓盈亏超过阈值
  └── 🟢 日常（定时）
      ├── 每日开盘前：持仓摘要 + 隔夜新闻汇总
      ├── 每日收盘后：当日成交汇总 + PDT 计数
      └── 每周：绩效报告 + 学习要点

  推送分级控制：
  - 可配置每种类型的推送开关
  - 非交易时段静默（可配置）
```

### 4.9 远程计算 (`scripts/remote/` + Makefile)

```
适用场景：
  - 大规模参数优化（网格搜索 / 贝叶斯优化）
  - 多年历史数据批量回测
  - 多策略并行对比

工作流：
  make sync          # 同步代码到 远程服务器
  make remote-backtest  # 在 远程执行回测
  make remote-fetch     # 取回结果文件

Makefile 关键命令：
  sync              - 同步代码（排除 .git/data/logs 等）
  remote-backtest   - 在 远程执行批量回测
  remote-optimize   - 在 远程执行参数优化
  fetch-results     - 从远程拉取结果到本地
```

## 六、关键设计决策回答

### Q1: 需要远程运算吗？

> **是的**。大规模回测和参数优化会涉及大量计算，通过 `make sync` 同步到 远程服务器执行，结果回传。日常轻量操作在本地完成。

### Q2: 通知推送到个人微信？

> **可以**。使用**企业微信机器人 Webhook**（免费，无需企业认证），将机器人添加到个人微信的群聊中即可接收推送。备选方案：Server酱。

### Q3: TWS 还是 IB Gateway？

> **选择 IB Gateway**。它比 TWS 更轻量（内存占用小得多），没有复杂的图形界面弹窗，适合程序化交易和长时间运行。TWS 仅在需要手动看图时使用。

### Q4: 账号信息存储在哪里？

> 敏感信息全部存储在 **`.env`** 文件中（不提交 Git）。包括：
> - IBKR 账户 ID、TWS/Gateway 端口
> - 企业微信 Webhook URL
> - 新闻 API Key
> `.env.example` 提供模板，实际值由你填写。

### Q5: 时政信息如何影响交易？

> 三阶段流程：
> 1. **获取**：RSS + API 多源获取财经/时政新闻
> 2. **分析**：FinBERT 模型做情感评分（-1~+1）
> 3. **调整**：高负面 → 提高风控等级 → 建议减仓/暂缓 → 微信推送告警
>
> 重大突发事件可触发紧急模式（允许 PDT 例外平仓）。

### Q6: 账户资金不足，PDT 限制？

> 账户 < $25,000 时受 PDT 规则限制：5 个交易日内 ≤ 3 次日内交易。
> **风控模块内置 PDT 计数器**：
> - 自动跟踪过去 5 个交易日的日内交易次数
> - ≤ 2 次：正常 + 提醒
> - = 2 次：橙色预警 + 二次确认
> - ≥ 3 次：硬阻止
> - 紧急情况例外通道（记录原因）

### Q7: 金融知识学习文档？

> `docs/` 目录按主题组织，从基础到进阶，每篇聚焦一个概念，结合实操。交易中遇到不理解的术语可以随时查阅。

### Q8: 交易记录存储？

> 所有交易记录存入 `data/trade.db`（SQLite），字段完备，支持多维查询和统计分析。

## 七、关键风险与应对

| 风险 | 应对 |
|------|------|
| IBKR API 2FA 限制 | 保持长连接，用 IB Gateway；无法完全无人值守启动 |
| Pacing Limit | 本地缓存 + 增量拉取，冷启动用 yfinance 补充 |
| PDT 误判导致违规 | 风控模块硬阻止 + 下单前检查 + 微信告警双重提醒 |
| FinBERT 噪声过高 | 只对高置信度（>0.7）结果发出调整建议；人工可覆盖 |
| 远程服务器无法联网 | 远程服务器为内网，回测数据需本地下好同步上去 |
| 实盘误操作 | 风控硬限制 + Paper 验证 30 天后再上线 |

#### 当前硬编码阈值（待量化优化）

| 检测项 | 当前阈值 | 问题 |
|--------|----------|------|
| 盘中急跌抄底 | 日内跌 > 3% | 未区分高波动/低波动股票 |
| 盘中急涨告警 | 日内涨 > 5% | 杠杆ETF轻松触发 |
| 跳空回落 | 开盘跳空 > 2% + 回落 > 1% | 阈值一刀切 |
| 日内低位反弹 | 反弹 > 3% | 绝对值，未用百分比归一化 |
| Z-Score极端 | > 2.5σ | 未区分不同市况 |
| 成交量异常 | > 3x 均量 | 未考虑盘前盘后低量 |
| 邮件告警触发 | 日间 > 3% 或 日内 > 4% | 凭经验设定 |

#### 计划引入的量化算法

| 算法 | 用途 | 说明 |
|------|------|------|
| **ATR 归一化** | 用 ATR 替代固定百分比 | 高波动股阈值自动放宽 |
| **布林带 %B** | 标准化超买超卖判断 | 比固定价格更好 |
| **Z-Score 动态窗口** | 自适应均线周期 | 短/中/长线分别计算 |
| **VWAP 偏离度** | 机构成本线偏离 | 识别主力意图 |
| **成交量分布 (Volume Profile)** | 识别关键价格区域 | 高成交量节点 = 支撑/阻力 |
| **订单流不平衡 (Order Flow)** | 买卖压力对比 | 需 Level 2 数据 |
| **卡尔曼滤波** | 降噪价格信号 | 统计物理背景直接可用 |
| **隐马尔可夫模型 (HMM)** | 识别市场状态切换 | 趋势/震荡/高波动的概率 |
| **Hawkes 过程** | 自激点过程建模极端事件 | 黑天鹅概率预估 |
| **Copula 相关性** | 多股票联动异动检测 | 板块轮动识别 |
