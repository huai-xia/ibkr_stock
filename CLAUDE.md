## ⚠️ 下单安全规则（极其重要）

### 缺失信息 → 必须追问

下单需要以下完整信息，**缺一不可**：

| 必须确认 | 参数 | 示例 |
|----------|------|------|
| 股票代码 | `<SYMBOL>` | AAPL, TSLA, NVDA |
| 买入/卖出 | `buy` / `sell` | buy |
| 数量 | `<QTY>` | 100 |
| 订单类型 | `--type` | limit / market / stop / stop_limit |
| 价格（限价单） | `--price` | 180.00 |
| 止损价（止损单） | `--stop-price` | 175.00 |
| 端口 | `--port` | 4001 / 4002 |

**如果用户说「买 NVDA」但没有说明：**
- 买多少股？→ **必须问**
- 限价还是市价？→ **必须问**
- 什么价格？→ **必须问**
- 真实账户还是模拟？→ **必须问**

### 必须给出金融建议

下单前，根据场景给出建议：

- **市价单**：⚠️ 提醒滑点风险，盘前/盘后禁止市价单
- **限价单**：✅ 推荐，解释价格设在哪更合理（如支撑位附近）
- **止损单**：✅ 推荐配合使用，给出建议止损位
- **资金占比**：提醒该笔交易占总资金百分比，超过 20% 预警
- **PDT 风险**：如果可能触发日内交易，提前警告

### 真实账户操作 → 反复确认

涉及 `--port 4002` + `--force` 的下单操作：

1. **第一次确认**：「你确定要用真实账户下单吗？」
2. **第二次确认**：「请确认以下信息无误：股票 XXXX，方向 XXXX，数量 XXXX，价格 XXXX」
3. **风控提醒**：执行前展示 PDT 状态和仓位占比
4. 用户明确回复「确认」「是的」后才能执行

### 不合理的操作 → 提醒并建议

| 情况 | 处理 |
|------|------|
| 单票超过总资金 20% | ⚠️ 提醒分散风险，建议降低 |
| 盘前/盘后用市价单 | 🚫 强烈劝阻，建议等开盘或用限价单 |
| PDT 只剩 1 次 | ⚠️ 提醒即将触发限制 |
| 没有止损 | ⚠️ 建议设止损位 |
| All-in 单只股票 | 🚫 劝阻，说明风险 |

## 回测规则

回测相对安全，可以更宽松地处理：

- 策略名：`momentum`（双均线动量）或 `mean_reversion`（布林带回归）
- 默认 365 天，可用 `--days` 调整
- 默认资金 $10,000，可用 `--capital` 调整

**如果用户没指定策略**，默认同时跑两个策略并对比结果。

## 市场时间感知

- 美东时间 9:30-16:00：常规交易时段
- 盘前 4:00-9:30 / 盘后 16:00-20:00：流动性差，提醒风险
- 周末/节假日：市场关闭，只能查看历史数据
- 用户说「现在」查行情时，如果市场已关闭，主动告知
